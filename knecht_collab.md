## 1. Guidelines from original proposal (high‑level)

From the proposal + spending history, the intent was essentially:

*   Improve **research discovery**, visibility, and collaboration
*   Support **library‑affiliated research tools**
*   Fund **technology, software, and people‑enabling infrastructure**
*   Benefit a **broad FAU audience**, not a single lab

That gives a lot of room for:

*   Events that *activate* tools
*   Training + facilitated collaboration
*   Situating the agent as shared research infrastructure

So framing everything as **“reducing friction between researchers”** is the right throughline.

## 2. Enhancement themes (conceptual buckets)

Instead of “what features can we add to the agent?”, think:

### A. From *Discovery* → *Connection*

The agent already answers:

> “Who at FAU might I collaborate with?”

The missing layer could be:

> “How do I actually meet them, and what happens next?”

### B. From *Tool* → *Catalyst*

The agent can potentially **trigger real‑world interactions**.

### C. From *Passive use* → *Facilitated moments*

Most researchers won’t explore on their own — but they *will* show up to a time‑boxed, guided experience.

## 3. Fundable ideas (in‑person/workshop‑oriented)

Below are ideas that are all:

*   Clearly aligned with the original proposal
*   Plausible uses of tech fee dollars
*   Scalable across disciplines

### 1. “Research Matchmaking” Workshop (flagship idea)

**What it is**

*   Small, curated, in‑person sessions (20–40 people)
*   Participants run Michelle’s agent **before or during** the workshop
*   Each person gets a short, “collaboration brief”:
    *   Their interests
    *   3–5 FAU researchers surfaced by the agent
*   Facilitated networking with structured prompts -- enhanced agent that we demo

**Funds used for**

*   Event facilitation
*   Space, food, printed materials
*   Light staffing / student support
*   Possibly modest stipends for facilitators or librarians

**Why it fits**

*   Tool → real collaboration
*   Library‑led
*   High visibility
*   Reusable format (medicine, engineering, humanities, etc.)

### 2. A “Collaboration Studio” Series (Multi‑session?)

**What it is**
A short series (e.g., 3 sessions):

1.  **Discover** – using the agent + FAU research data
2.  **Define** – refine collaboration ideas with librarians + research computing
3.  **Pitch** – lightning talks / posters among participants

**Funds used for**

*   Series coordination
*   Light tech setup (screens, kiosks)
*   Food + logistics
*   Documentation (outputs can be reused)

**Strategic benefit**

*   Makes the agent part of a *process*, not a novelty
*   Generates artifacts leadership loves (posters, summaries, metrics)

### 3. Hackathon Adjacent: “Agent‑Powered Networking Zones”

Since you have a nebulaONE hackathon coming up…

**Instead of just a student event**, carve out:

*   A **Research Collaboration Lounge**
*   Staffed by:
    *   A librarian
    *   An owlONE admin
*   Live demos:
    *   “Bring your research idea, we’ll show you collaborators”
*   Scheduled mini‑sessions throughout the day

**Funds used for**

*   Physical setup
*   Staffing
*   Signage
*   Small incentives

**Why this is smart**

*   Uses the hackathon’s energy
*   Broadens the audience beyond coders
*   Makes the agent visible as campus infrastructure

### 4. “Collaboration Seed Clinics” (Low‑stakes, high value)

**What it is**

*   Not grant funding — *pre‑grant*
*   Short, themed clinics (e.g., “AI + Health”, “Climate + Policy”)
*   Agent used to:
    *   Surface cross‑college connections
    *   Break silos intentionally

**Funds used for**

*   Event operations
*   Facilitation
*   Materials
*   Follow‑up documentation

You never promise money — just clarity and connections.

### 5. Pop‑Up Research Kiosks (Surprisingly effective)

**What it is**

*   Temporary stations in library / common areas
*   iPad or laptop running the agent
*   “Find your next collaborator in 5 minutes”
*   Staffed during peak hours

**Funds used for**

*   Hardware (if needed)
*   Signage
*   Part‑time staffing
*   Light marketing

This works especially well for people who would *never* attend a workshop.
